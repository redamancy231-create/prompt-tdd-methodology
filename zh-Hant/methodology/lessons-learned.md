## 三個跨實驗模式

### P1: 天花板效應是 prompt 實驗的沉默殺手

**觀察**: A2（程式碼審查）和 A3（路由決策）都在 GPT-5.5 temp=0 下遇到天花板——模型表現太好，效應量無法檢測。

**機制**: 當 baseline 準確率 > 0.9 時，Δ 的檢測空間被壓縮到 < 0.1。需要遠超常規的樣本量或更粗糙的 DV 才有統計顯著性——但更粗糙的 DV 本身會丟失資訊。

**關鍵資料**: A2 correctness_rate A=0.954, B=0.935, Δ=0.019, n_nonzero=4<5（科學門 FAIL）。A3 兩個實驗均 Δ=0, discordant 率=0%。

**教訓**: 實驗設計階段必做 ceiling probe（CK6）。如果探測顯示天花板不可打破，調整 DV 或任務域——不要指望"大樣本就能檢測到"。

### P2: IV 汙染比統計方法錯誤更危險

**觀察**: 對照實驗的核心威脅不是統計方法選擇（Wilcoxon vs t-test），而是 IV（prompt 格式）被資訊量差異汙染。

**A2 關鍵發現**: B 組（prep/exec/post 分段）實際包含了 A 組（一體式編號列表）沒有的檢查維度——不是因為格式不同，而是因為資訊量不等價。

**教訓**: CK4 INVENTORY 等價協議不是可選檢查——它是防止"測量了錯誤的東西"的唯一手段。如果兩組 prompt 的產出清單不等價，實驗結論無效，無論 p 值多少。

### P3: 陰性結果 ≠ 實驗失敗

**觀察**: A2 和 A3 科學門全部 FAIL——但產出了 21 個方法論片段、3 個實驗設計模板、一套可複用的分析管線。

**教訓**: 陰性實驗的價值不在"支援 H1"，而在：(a) 排除了特定條件下的格式效應存在性；(b) 暴露了實驗設計的系統性盲點（天花板、DV 選擇、IV 汙染）；(c) 積累了"如何不做一個錯誤實驗"的操作知識。

**誠實標註**: 兩個實驗成熟度均標 [E-]（ceiling-limited），不是 [E]——但仍然是可引用的有效證據。

## 六個缺失模式（PM-1~PM-6，來自原始復盤 §4.3 對抗驗證）

以下缺失模式是 A2+A3 復盤經 Codex GPT-5.5 魔鬼代言人式對抗驗證後識別的——不是"實驗做錯了什麼"，而是"系統性盲點：下一次實驗應該補什麼"。

| 編號 | 模式 | 定義（原始報告原文） |
|------|------|---------|
| **PM-1** | 模型-溫度組合作為普適混淆變數 | 兩個實驗均僅用 GPT-5.5 temp=0——格式效應的陰性結論不能推廣到其他模型/溫度。**經 Qwen 跨模型復現部分緩解**（A2 方向一致，Δ=−0.014），但審查行為/方法論傳遞/等價性驗證等維度仍僅 GPT-5.5 單點 |
| **PM-2** | 實驗者方法論成熟度作為跨實驗的向上偏誤 | A2 和 A3 的實驗者（同一個人/團隊）在兩實驗中方法論成熟度持續提升——A3 的設計品質高於 A2。這對"跨實驗模式"聲稱構成混淆：觀察到的改進是方法學成熟還是真有跨域規律？ |
| **PM-3** | 探索性 vs 確認性框架的持續模糊 | Tier 0 本質是探索性（生成假設），Tier 1 是確認性（檢驗假設）——但實驗報告未明確區分兩者的推斷邏輯和結論語言 |
| **PM-4** | 測試集規模×效應量檢測的靈敏度關係 | 未分析測試集規模和專案區分度對統計功效的聯合影響——只知道"n=4 不夠"，但不知道"需要多少" |
| **PM-5** | 評分/分析工具自身的實現有效性需要預審 | A3 的評分 bug（acceptable_paths 雙重使用 + meta action 懲罰）在收集後才被發現——評分腳本應該和實驗設計一樣經過預審 |
| **PM-6** | 獨立審查自身的誤判風險未被充分討論 | 復盤經 10 輪異後端審查——但審查者之間的分歧、審查衰減、審查者的系統性盲點本身未被分析。審查不是神的視角 |
