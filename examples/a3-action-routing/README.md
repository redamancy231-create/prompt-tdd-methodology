# A3: 声明式 vs NL 路由描述对照实验

> **角色**: **反例案例**——如何诚实闭合一个无法继续榨出信号的实验  
> **状态**: CLOSED [E-]（实验设计未能检验假设，而非假设被证伪）  
> **审查**: 10 轮（Codex GPT-5.5 ×5 + Qwen3.7-Max ×3 + 合并/对齐 ×1 + 咨询 ×1），0 未闭合发现

---

## ⚠️ 读前须知

A3 的尸检结论是：**不是假设被证伪，而是实验设计未能检验假设**。

- 天花板效应（GPT-5.5 在路由任务上接近完美）
- 样本量不足（Pilot 15 cases，95% 置信上界 ~18%）
- GT 定义争议（"正确路由"的操作化缺乏外部锚点）
- DV 退化（二分类 accuracy 对格式效应不敏感）

A3 的**教学价值**不在于"证明了什么"，而在于展示了一个实验在什么时候应该闭合、如何诚实记录闭合原因、以及如何从"失败"中提取方法论资产。

---

## 实验设计

| 参数 | 值 |
|------|-----|
| 任务域 | Agent 路由决策 |
| 研究问题 | 声明式结构化路由描述是否优于 NL 紧凑描述？ |
| A 组（对照） | NL 紧凑路由描述（长度控制） |
| B 组（实验） | 声明式结构化路由描述 |
| 主 DV | strict accuracy（0/1，含 meta action 豁免） |
| 模型 | GPT-5.5 temp=0 (Codex CLI) |

### 实验历程

| 阶段 | 样本 | 结果 | 决策 |
|------|------|------|------|
| v1 | n=10/臂，6 actions | Δ=0 | 加 Hard Mode（增加歧义） |
| Hard Mode | n=10/臂 | Δ=0 | Discordant 率 = 0% |
| Pilot | 15 cases，15 actions | Δ=0/30 calls | 评估三方案后选择闭合 |

### A3 v2 设计（未执行）
经 Codex + Qwen 双后端审查发现 DV 退化 + 格式×数量混淆，不执行。

---

## 结果

### 核心数据
- 三个阶段 Δ=0，discordant 率 = 0%
- Pilot 95% 置信上界 ~18%（Rule of 3）
- 评分 bug（acceptable_paths 双重使用 + meta action 惩罚）→ 修正后不影响结论

### 科学门: FAIL

原因不是"H1 被证伪"，而是"实验设计未能创造检测 H1 的条件"：
1. 天花板太高——GPT-5.5 在路由任务上接近完美
2. DV 不敏感——二分类 accuracy 无法区分格式效应
3. GT 无外部锚点——"正确路由"由实验者定义，没有独立验证

---

## 关键教训

1. **不是所有实验都能检验假设**：当实验设计无法创造检测空间时，继续追加样本是浪费——诚实地闭合比"做出显著结果"更有价值
2. **尸检比结果更重要**：A3 的价值不在 Δ=0，而在识别出 4 个根因（天花板 / DV 选择 / GT 缺失 / underpowered）
3. **Pilot 的止损价值**：A3 v2 设计经审查发现 DV 退化——如果 v2 被执行了，浪费的是 5 倍的 API 调用 + 评分 + 审查时间
